# S2E8: Multi-Agent Systems: Collaboration and Scaling Laws

## Summary
This episode examines whether multi-agent AI systems — where multiple AI agents coordinate, divide labor, and pass results to one another — actually outperform a single well-equipped agent. The answer, grounded in two recent research papers, is nuanced: multi-agent systems introduce real communication and coordination costs that can make them worse than a single agent, not just marginally but dramatically so.

The first paper (Davidson et al., 2025, "the collaboration gap paper") asks a foundational question: can AI agents actually collaborate with each other in an unstructured setting? Using a maze-sharing experiment, the researchers found that many models that perform well solo fall apart when required to coordinate with another agent — because they fail to establish a shared representation of the problem. Distilled models were especially vulnerable. A partial fix ("relay inference") has one strong agent set the terms of communication and a weaker one follow, but this is leader-follower, not true collaboration.

The second paper (Google DeepMind and MIT, 2024) studies scaling laws for multi-agent systems across 180 configurations. Three empirical patterns emerge: (1) tool-heavy tasks heavily penalize multi-agent overhead; (2) once a single agent can solve a task ~45% of the time, adding agents hurts more than it helps; and (3) hierarchical architectures amplify errors as mistakes propagate up the chain. The exception is genuinely parallelizable tasks with independent subtasks, where multi-agent setups can improve performance by over 80%. The episode ties the two papers together: the relay inference fix from paper one looks structurally like the sequential pipelines paper two flags as most fragile. The overall message — matching architecture to task is non-trivial, and a single good agent with good tools is often the right answer.

## Key Concepts
- **Multi-agent systems**: Networks of AI agents that coordinate, specialize, and pass results to each other to solve a task → see also: concepts/multi-agent-systems.md
- **Collaboration gap**: The finding that strong solo task performance does not predict strong collaborative performance in AI agents → see also: concepts/collaboration-gap.md
- **Shared representation problem**: Agents failing to agree on a common language or structure for describing a task, causing coordination to break down → see also: concepts/shared-representation.md
- **Relay inference**: An architecture where a strong agent goes first and establishes the communication frame, with a weaker agent following its lead → see also: concepts/relay-inference.md
- **Model distillation**: Compressing a large model into a smaller one; discussed here because distilled models showed the most severe collaborative failure despite solid solo performance → see also: concepts/model-distillation.md
- **45% threshold**: An empirical finding that multi-agent systems stop helping and start hurting once a single agent can solve the task ~45% of the time → see also: concepts/multi-agent-scaling-threshold.md
- **Error amplification in hierarchies**: In hierarchical agent topologies, errors made by sub-agents propagate upward and corrupt downstream reasoning → see also: concepts/error-amplification-hierarchies.md
- **Task parallelizability**: The degree to which a task can be split into independent subtasks; the key determinant of whether multi-agent systems add value → see also: concepts/task-parallelizability.md

## Takeaways
1. **Collaboration is a distinct capability from raw intelligence.** A model that is excellent at solving a task alone may be poor at explaining its reasoning in a way another agent can use — these are separable skills that current training does not jointly optimize.
2. **The 45% threshold is a practical decision rule.** If a single agent already solves your task roughly half the time on its own, adding agents is likely to make things worse due to coordination overhead — save multi-agent complexity for harder, lower-baseline tasks.
3. **Distilled models are particularly fragile collaborators.** Distillation preserves answer quality but strips away the explanatory clarity that makes reasoning communicable to another agent; this is an underappreciated cost of model compression.
4. **Task structure, not agent count, drives multi-agent success.** Genuinely parallelizable tasks (independent subtasks, no sequential dependencies) can see 80%+ gains; sequential tasks with chained dependencies saw 70% degradation in the same framework.
5. **The relay inference fix trades one problem for another.** Having a strong agent set the communication frame helps collaboration, but the resulting sequential handoff is exactly the topology the scaling laws paper identifies as most prone to error propagation.
