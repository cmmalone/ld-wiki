# S2E1: What Is an AI Agent?

## Summary
This season-opening episode sets up an eleven-episode arc dedicated to unpacking AI agents — what they are, how they work, where they fail, and what it means to delegate real judgment to one. Rather than diving straight into technical detail, the episode earns its footing by working through a definitional problem that turns out to be harder than it sounds: the phrase "AI agent" is used to describe everything from a slightly smarter chatbot to a system autonomously managing your email inbox, and that ambiguity isn't just sloppy marketing. It reflects something genuinely slippery about the concept itself.

The episode builds a working definition through four increasingly complex examples. A basic ChatGPT interaction — ask a question, get an answer — is labeled a "responder": nothing in the world changes, there's no loop, no persistence. Adding a web search tool gets closer, but if the model searches once and stops, it still isn't really an agent. The flight-booking example is where agency proper begins: the model must search, compare results, notice when criteria aren't met, try again, fill forms, and know when it's done — all driven by a continuous observe-reason-act cycle, and all with real financial and logistical consequences. The fourth example, an email-managing agent running continuously in the background, pushes into philosophical territory about delegation, authorization, and accountability.

The working definition adopted for the season comes from the ReAct paper: an agent is something running a loop of observation, reasoning, and action — with real consequences on the other end. This framing is chosen not because other definitions (Russell & Norvig's sensor/actuator model, Anthropic's consequences-first definition) are wrong, but because the loop is what makes agents uniquely interesting and hard, and it will be the through-line for everything that follows.

## Key Concepts
- **AI Agent (working definition)**: A system running a continuous observe-reason-act loop with real-world consequences, as opposed to a one-shot responder. → see also: concepts/ai-agent.md
- **Observe-Reason-Act loop**: The core cycle from the ReAct paper in which a model perceives the result of its last action, decides what to do next, and acts again — the distinguishing feature of agency. → see also: concepts/react-loop.md
- **Tool use**: The ability of an LLM to reach outside itself and invoke external capabilities (e.g., web search, form submission), a necessary but not sufficient condition for agency. → see also: concepts/tool-use.md
- **Agent accountability**: The unresolved question of who is responsible when an agent takes a consequential action — booking a ticket, sending an email — that turns out to be wrong. → see also: concepts/agent-accountability.md
- **Policy delegation**: The mechanism by which a user hands an agent a set of standing rules or criteria (e.g., "accept coffee invitations under these conditions") and then steps out of the loop entirely. → see also: concepts/policy-delegation.md
- **Multi-agent systems**: Systems in which multiple agents coordinate, divide work, or communicate with each other — flagged here as an upcoming topic. → see also: concepts/multi-agent-systems.md
- **Agent evaluation**: The challenge of assessing agent performance given the complexity and branching nature of multi-step, real-consequence tasks — flagged as a forthcoming episode topic. → see also: concepts/agent-evaluation.md

## Takeaways
1. **Using a tool once isn't agency** — what matters is whether the model observes the result of that action and decides what to do next. The loop is the thing.
2. **Consequences are a definitional feature, not a side effect**: a true agent produces outcomes in the world (a charged card, a sent email) that can't always be caught or reversed when wrong.
3. **The hardest design question in agents isn't capability — it's knowing when to stop**: determining that a goal has been met is a genuinely open problem, not a solved one.
4. **Delegation exists on a spectrum**: from a discrete task (book this flight) to ongoing judgment (manage my email forever), and the further along that spectrum you go, the harder the accountability questions become.
5. **The host built a real agent for this podcast**: it cleans transcripts, generates newsletters, and will be dissected in the season finale — grounding the whole series in a concrete, working artifact.
