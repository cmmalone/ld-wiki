# S2E5: Agentic Planning

## Summary

This episode explores how AI agents can move beyond reactive, step-by-step behavior toward genuine planning — the ability to look ahead, explore alternative paths, and backtrack when something isn't working. The central exhibit is the 2023 paper *Tree of Thoughts*, which demonstrated that the same underlying model (GPT-4) could jump from solving 4% of "Game of 24" puzzles using chain-of-thought prompting to solving 74% using a structured tree search. The key reframe: planning is fundamentally a search problem, not just a prediction problem.

The episode carefully distinguishes between two things that can sound similar but aren't — memory (what an agent can access) and planning (what an agent can *do* with it). Standard LLM generation and even chain-of-thought prompting are characterized as "system one" thinking: reactive, linear, and committed to a single path. Tree of Thoughts imposes "system two" reasoning by generating multiple candidate next steps at each point, having the model evaluate which branches look promising, pruning dead ends, and backtracking when needed. Crucially, the same model can serve as both the generator of candidate steps and the evaluator of their promise — and empirically, evaluation turns out to be an easier task than generation.

The episode closes with a practical tension: Tree of Thoughts is expensive, combinatorially so, and most production agents today operate in a cheaper reactive mode even when planning would serve them better. The emerging research direction involves agents that start with chain-of-thought and escalate to tree search only when they detect they're stuck — a kind of adaptive planning. The episode also draws a line to modern "reasoning models" (OpenAI's o-series, Claude's extended thinking) as systems that have learned this planning behavior natively rather than having it imposed by prompting.

## Key Concepts

- **Reactive agents (think-act-observe loop)**: The default agent architecture cycles through perceiving state, choosing a next action, and observing the result — with no explicit lookahead or branching → see also: concepts/think-act-observe-loop.md
- **Chain-of-thought prompting**: A prompting technique that asks the model to reason step-by-step before answering; externalizes reasoning but commits to a single linear path with no backtracking → see also: concepts/chain-of-thought-prompting.md
- **Tree of Thoughts**: A 2023 planning algorithm that structures LLM reasoning as a tree search — generating multiple candidate steps, evaluating branches, pruning dead ends, and backtracking — lifting Game of 24 solve rate from 4% to 74% → see also: concepts/tree-of-thoughts.md
- **Game of 24**: A combinatorial math puzzle used as a benchmark; requires genuine search rather than pattern matching, making it a clean test for planning capability → see also: concepts/game-of-24-benchmark.md
- **System 1 vs. System 2 thinking**: From Kahneman's *Thinking Fast and Slow* — System 1 is fast and reactive (raw LLM generation), System 2 is slow and deliberate (Tree of Thoughts); the episode frames planning as an attempt to give LLMs genuine System 2 capability → see also: concepts/system1-system2-thinking.md
- **Problem space search**: A framework from Newell and Simon (1950s) modeling cognition as search through a graph of possible states; Tree of Thoughts explicitly revives this as a lens for LLM planning → see also: concepts/problem-space-search.md
- **Planning vs. memory in agents**: A conceptual distinction — memory is about what an agent can access; planning is about whether it can model paths forward, explore alternatives, and adjust mid-task → see also: concepts/agentic-memory-vs-planning.md

## Takeaways

1. **Planning and memory are different problems.** Memory is about what an agent can hold or retrieve; planning is about whether it can reason about paths forward, explore alternatives, and know when to backtrack. Conflating them misidentifies where agents break down.

2. **Evaluation is easier than generation.** The same model that struggles to generate the correct next step can often reliably judge whether a partial solution is on the right track — just as checking a Game of 24 answer is easier than finding one. This asymmetry is what makes self-guided tree search work at all.

3. **Chain-of-thought fails at combinatorial problems by design.** It has no backtracking mechanism and commits confidently to a single path. For tasks where the right answer isn't on the most plausible-looking route, chain-of-thought will fail fluently and confidently.

4. **The capability exists; the question is when to pay for it.** Tree of Thoughts works, but it's expensive enough that most production agents don't use it by default. When agents disappoint in real-world tasks, it may not be a fundamental model limitation — it may be a cost-driven decision to stay in reactive mode.

5. **The boundary between prompted planning and learned planning is dissolving.** Models like OpenAI's o-series and Claude's extended thinking appear to have internalized something like tree search natively, rather than requiring it to be imposed via prompting — suggesting the field is moving toward planning as a trained capability, not just a scaffolding trick.
