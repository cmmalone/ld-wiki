# S2E6: AI Agent Failure Modes and Benchmarks

## Summary
This episode serves as a reality check on AI agents, examining the mathematical and taxonomic reasons why agents fail even when their per-step accuracy seems high. The core mathematical insight is deceptively simple: compound probability. An agent that is 90% reliable at each step achieves only ~35% end-to-end success on a 10-step task, and rounds to 0% on a 100-step task. This means the reliability bar required for each individual step is far higher than most current systems can meet — you need four nines (99.99%) per-step reliability just to clear 90% end-to-end on a 100-step task.

Beyond the math, the episode surveys benchmark evidence from TAU-Bench (customer service simulations) and the SWE-bench family (software engineering), showing a split picture: performance on narrow, well-defined tasks has improved dramatically (from ~2% to ~94% on SWE-bench Verified since 2023), while performance on broader, more realistic tasks remains much lower (12% on SWE-bench Mobile). The TAU-Bench Pass@K metric makes the reliability problem vivid — even well-performing models succeed at a task all eight consecutive times less than 25% of the time.

The episode closes with a taxonomy of agent failure modes from the MAST paper (UC Berkeley, 2025), which organizes 14 failure modes into three categories: specification/design failures, inter-agent misalignment, and task verification/termination failures. A counterintuitive finding: multi-agent systems don't reliably outperform single-agent systems, because coordination introduces an entirely new failure category without guaranteeing proportional gains. The episode frames both agent capability improvement and user skill development as jointly responsible for the wide variance in real-world user experiences.

## Key Concepts
- **Compound error probability**: In a multi-step system, per-step error rates multiply, causing end-to-end reliability to degrade sharply as task length grows → see also: concepts/compound-error-probability.md
- **Error propagation in agents**: A mistake at step N corrupts the world state that all subsequent steps reason from, often invisibly to the agent → see also: concepts/error-propagation.md
- **Pass@K metric**: A reliability metric from TAU-Bench that measures whether an agent can successfully complete the same task K consecutive times → see also: concepts/pass-at-k.md
- **TAU-Bench**: A benchmark simulating agentic customer service tasks (retail and airline) with tool access and a simulated user → see also: concepts/tau-bench.md
- **SWE-bench / SWE-bench Verified**: A benchmark of real GitHub software engineering issues used to evaluate coding agents → see also: concepts/swe-bench.md
- **MAST (Multi-Agent System Failure Taxonomy)**: A UC Berkeley 2025 taxonomy of 14 agent failure modes across three categories, derived from 1,600+ execution trace annotations → see also: concepts/mast-taxonomy.md
- **Inter-agent misalignment**: Failures specific to multi-agent systems arising from handoff errors, contradictory conclusions, and ambiguous orchestrator instructions → see also: concepts/inter-agent-misalignment.md
- **Agent scaffolding**: The tools, memory systems, and orchestration architecture surrounding an LLM that can produce up to 6x performance variance independent of the model itself → see also: concepts/agent-scaffolding.md

## Takeaways
1. **The math is brutal**: A 90%-per-step agent on a 100-step task has essentially 0% end-to-end success. Four nines of per-step reliability are required to achieve 90% end-to-end on a 100-step task — a bar no current system meets.
2. **Benchmarks tell different stories depending on scope**: SWE-bench Verified (~94%) and SWE-bench Mobile (~12%) are both "coding benchmarks" using frontier models, but the 8x difference reveals how much task breadth and realism matter.
3. **Multi-agent ≠ more reliable**: Adding agents adds an entire failure category (inter-agent misalignment) that doesn't exist in single-agent systems, and the coordination overhead can outweigh the specialization benefits.
4. **Doing it once vs. doing it consistently are very different problems**: Pass@K numbers drop sharply as K increases, revealing that headline benchmark scores often hide significant inconsistency.
5. **User skill matters as much as model capability**: The gap between "agents changed my life" and "agents are garbage" partly reflects how much users have learned to scope tasks, prompt effectively, and identify where agents are and aren't capable.
